1. XSSR: Cross-Domain Self-Supervised Representative Selection for Efficient Annotation in Medical Image Segmentation

背景

论文明确指出，跨设备、跨中心的医学分割里，真正昂贵的往往不是模型，而是目标域标注预算。它把这一问题放到眼底场景中讨论，数据里直接包含 RIGA+ retinal fundus imaging，因此对眼底迁移很贴近。

模型 / 算法

方法流程是先在无标注源域上训练 Masked Autoencoder 建共享 embedding，再用密度-新颖性-多样性联合评分对目标域样本做贪心选择，最后只用被选中的子集训练 U-Net。流程图明确表达了从“无标注源域表征”到“目标样本选择”再到“目标域分割训练”的闭环。

训练方法

论文明确写出固定 5% 标注预算，并自动调节 novelty-diversity 权重。训练阶段采用先表征学习、后样本选择、再分割训练的两阶段策略，核心目的不是把所有目标样本都标注，而是把有限预算花在最有代表性的样本上。

数据来源

数据覆盖 Chest X-ray、RIGA+ retinal fundus imaging 和 multi-site Prostate MRI。对眼底任务最关键的是 RIGA+，但论文也用其他医学模态说明该策略不是只对单一数据集有效。

具体创新点

论文把“选哪些目标样本去标注”显式变成可优化问题，而不是依赖经验或纯随机采样。我的判断是，它的关键不是 MAE 本身，而是把 MAE embedding 和代表性评分组合起来，形成了一个完整的样本选择闭环。

实验设计

论文在 Chest X-ray、RIGA+ 和 Prostate MRI 上做比较，核心对比是随机选择与 CoreSet 等代表性采样方法。图示和结果段落共同说明，它不是只看最终 Dice，而是看在固定预算下能维持多少分割性能。

实验结果

论文明确写到，在 Chest X-ray 上只用 22 张标注图就达到全量性能的 99.3%；在 Prostate MRI 上比随机选择高最多 2.5 Dice；整体比 CoreSet 高 0.4 到 1.2 Dice。这里的代价是它仍然依赖源域 embedding 的质量，且 fundus 只在 RIGA+ 上验证。

局限

论文明确指出，方法依赖源域 embedding 的质量，源域表征如果不稳，后续代表性选择就会偏。它还只在 RIGA+ 这个眼底数据上验证，跨更复杂眼底设备和病种的外推仍未完全打开。

对眼底分割的可用性

这篇最适合迁移到“标注预算很小但目标域很多”的眼底多中心分割场景，比如新医院、新相机或新病种的优先标注排序。落地上需要补的是更强的 fundus 预训练表征，以及把选择策略和眼底病灶/血管结构知识结合起来。

审稿式判断

方法优点

论文明确指出了一个闭环问题：不是“怎么训更大模型”，而是“怎么在有限标注下选最该标的样本”。我的判断是，它的优点在于流程完整，选择准则可解释，而且实验中确实在固定预算下跑出了稳定收益。

方法缺陷

方法依赖源域 MAE 表征的好坏，这意味着源域分布如果偏，后续 sample scoring 也会跟着偏。另一个问题是，贪心选择的收益虽然清楚，但对超大规模目标域时的计算和调参成本未明确说明。

适合复现的理由

代码结构上应该比较清晰：MAE 预训练、样本评分、贪心选择、U-Net 训练是四个独立模块。数据也相对容易拿到，尤其是 RIGA+ 这类眼底集合，使得复现主结果的门槛不算高。

适合改造的理由

最适合插入眼底分割流水线的是“样本选择”这一步，可以替换掉随机采样或纯不确定性采样。我的判断是，它也适合和细血管优先标注、病灶优先标注结合，做成更符合眼底任务的主动学习框架。

2. Cross-Domain Vessel Segmentation via Latent Similarity Mining and Iterative Co-Optimization

背景

论文明确把问题定义为视网膜血管分割的跨域掉点，根因是采集设备、成像风格和对比度差异过大。对眼底来说，这个问题尤其现实，因为不同医院和相机的域偏移会直接破坏血管边缘和连通性。

模型 / 算法

方法是先预训练源域与目标域的生成网络，再通过条件扩散模型做 deterministic inversion，把血管图像映射到中间 latent，随后在 latent 上构造域无关 prototype，并与分割网络迭代共优化。流程图明确展示了生成网络、潜空间挖掘和分割网络之间的相互拉动关系。

训练方法

论文明确写到生成与分割是 iterative co-optimization，也就是不是先训完生成模型再单独训分割器，而是两者循环更新。这样的训练策略意图是让 latent similarity mining 不只反映图像外观，而是反映真正可分割的血管结构。

数据来源

摘要和正文片段都明确指向 retinal vessel 场景，但未在可见文本中展开全部 benchmark 名称。我的判断是，它重点面向跨域或跨模态的临床血管分割，而不是单一公开基准的局部优化。

具体创新点

论文的核心创新是把跨域血管相似性挖到 latent 层，再让生成和分割双网络协同修正。与直接做特征对齐相比，这种做法更强调结构层面的 prototype 共享，而不是只对齐表面特征。

实验设计

论文摘要明确说它在存在显著 modality discrepancies 的临床场景里验证，并以 SOTA 作为主要结论。正文可见材料没有展开具体的对比方法和消融表，因此这部分信息未明确说明。

实验结果

论文明确声称在跨域临床场景中达到 SOTA，但没有在可见摘录里给出完整数值。我的判断是，它的结果更像“临床跨域有效”的方向性证据，而不是可以直接复现数值的轻量基线。

局限

方法链条比较重，涉及生成模型、扩散反演和迭代优化，训练稳定性和推理成本都会更高。另一个限制是可见文本没有展开完整 benchmark 和消融，说明其结果透明度还不够高。

对眼底分割的可用性

这篇最适合迁移到多中心眼底血管分割，尤其是设备差异大、图像风格变化明显的场景。落地难度偏高，但如果目标是做更强的跨域鲁棒性，它提供了一个比普通特征对齐更激进的路线。

审稿式判断

方法优点

论文明确把“潜空间相似性”作为跨域核心，这比只做像素或特征层对齐更贴近结构分割任务。我的判断是，它的优点在于理论链条完整，而且针对 domain shift 的处理不是附会式增强，而是嵌入到生成-分割闭环里。

方法缺陷

依赖生成式管线，意味着算力和工程复杂度都明显上升。论文可见部分也没有把训练稳定性、收敛代价和实际部署成本讲透，这会影响后续工程化落地。

适合复现的理由

如果公开代码足够完整，这类方法有清晰的模块边界，理论上可复现流程。但我的判断是，真正复现主结果的难点不在分割器，而在扩散反演和迭代协同训练的稳定实现。

适合改造的理由

它特别适合改造成眼底跨域的结构对齐模块，尤其可以替换掉普通的 domain alignment loss。也可以进一步和细血管优先建模结合，专门强化低对比、边缘断裂的恢复。

3. Effect of Input Resolution on Retinal Vessel Segmentation Performance: An Empirical Study Across Five Datasets

背景

论文明确指出，很多 fundus pipeline 会先缩放图像，但细血管在这一步可能被压成亚像素结构。对眼底分割来说，这不是细节问题，而是会直接影响后续血管连通性和小血管召回。

模型 / 算法

方法以 UNet 为基线，在多种 downsampling ratio 下重复训练，并引入 width-stratified sensitivity，把血管按 thin、medium、thick 分开评估。论文的图示和实验设计共同说明，它不是在比复杂模型，而是在量化输入分辨率本身的影响。

训练方法

论文明确写到只改变输入分辨率，尽量固定其他设置，以确保结论可归因于缩放策略。这样的控制变量设计很适合回答“到底是不是分辨率伤害了细血管”这个具体问题。

数据来源

论文使用 DRIVE、STARE、CHASE_DB1、HRF 和 FIVES 五个公开数据集。这个覆盖面足够说明结论并非只对单一数据集成立。

具体创新点

它把“分辨率”提升为可量化变量，并给出细血管专用指标，而不是只用整体 Dice 盖过去。我的判断是，这种做法最大的价值在于把一个常被忽略的预处理环节变成了可讨论、可比较、可复现的实验对象。

实验设计

论文在五个数据集上扫多种缩放倍率，并重点观察 thin vessel sensitivity 的变化。它还显式比较 Dice 和细血管敏感度，说明主结果并不只看整体平均指标。

实验结果

论文明确写到，高分辨率数据集适度下采样更利于 thin vessel sensitivity；在 DRIVE 上过度下采样会让 thin vessel sensitivity 最高下降 15.8 个百分点，而 Dice 变化不大。这个结果非常关键，因为它说明常规 Dice 会掩盖小血管损伤。

局限

论文只用 UNet 作为基线，对 Transformer 或多尺度专用网络的外推需要谨慎。我的判断是，它给出的不是“所有模型都该怎么做”的终局答案，而是一个非常重要的分辨率敏感性证据。

对眼底分割的可用性

这篇最适合直接指导眼底数据预处理和训练输入规范，尤其是要保住小血管和末梢连通性的场景。落地上最现实的改造方向，是把分辨率选择写进训练协议，而不是当成默认预处理一步带过。

审稿式判断

方法优点

论文明确指出了一个容易被忽略但很关键的系统性问题，并用控制变量实验把它讲清楚了。我的判断是，它的优点不在复杂方法，而在证据很干净，结论对眼底分割非常可执行。

方法缺陷

方法本身不提出新的分割架构，所以它解决的是“怎么喂数据”的问题，而不是“怎么提升上限”的问题。另一个缺点是只用 UNet 做主实验，模型外推还不充分。

适合复现的理由

复现难度低，因为核心就是不同分辨率下重训同一模型。数据集公开、训练目标直接、结果指标明确，适合作为眼底分割的实验基线复核。

适合改造的理由

最适合改造的是训练数据入口和评估协议，可以把 thin vessel sensitivity 直接纳入你的训练/验证流程。我的判断是，它还能和超分辨率、patch crop、细血管增强策略联动，形成更完整的输入策略。

4. MS-DKC: A Dataset Knowledge Card Framework for Designing and Adapting Medical Image Segmentation Models

背景

论文明确提出，传统医学分割常常是先选模型，再看结果，但更合理的问题应该是：这个数据集到底要求什么样的模型能力。对眼底分割来说，这个问题尤其重要，因为 DRIVE 这类数据的结构偏好和其他病种完全不同。

模型 / 算法

方法把数据集特征分成 image/acquisition、morphology、supervision、context-dependence 和 deployment-risk 五类，再把这些证据映射到 failure modes、design priors 和 risk-aligned criteria。流程图说明它不是单一网络，而是一套从数据证据到模型决策的框架。

训练方法

论文强调 validation-constrained score-function selection 和风险对齐，而不是盲目扩大模型。换句话说，它把训练和模型选择都绑定到数据集知识卡上，而不是让模型自己去“碰运气”。

数据来源

论文明确使用 DRIVE、ISIC2018 和 ACDC。对眼底而言，DRIVE 是最直接的证据，说明这套框架不是纯方法论空转，而是已经落到了具体眼底数据集上。

具体创新点

它把数据集知识从隐性经验变成显式卡片，这一点对细结构任务尤其有用。我的判断是，它真正的价值不在某个新模块，而在把数据集的成像、监督和部署风险系统化地前置。

实验设计

论文在 DRIVE、ISIC2018 和 ACDC 上分别给出模型变体和对比结果，并报告 Dice、IoU、AUC 等指标。它还把设计选择与风险约束连在一起，所以实验不只是展示分数，还在解释为什么某种设计更适合某个数据集。

实验结果

论文明确给出 DRIVE 上 DKC-TNet-v2 Dice 0.8044；SA-UNetv2-DKC-AmbRef Dice 0.8141、IoU 0.6865、AUC 0.9853；ISIC2018 上 MS-DKC-AttNextTopo-VCSF-NoAug Dice 0.8872。结果说明知识卡框架至少在公开数据上是有效的，不是纯概念设计。

局限

论文更像一套结构化方法论，而不是即插即用的新 backbone。我的判断是，真正落地时需要人工整理数据集知识卡，这会增加前期建模成本。

对眼底分割的可用性

这篇很适合用于眼底分割项目立项前的任务分析，尤其适合判断该优先做边界、连通性还是域适配。落地难度不在实现，而在把知识卡做得足够准确、足够一致。

审稿式判断

方法优点

论文明确给出一个很强的思路：数据决定模型，而不是模型倒推数据。我的判断是，它的优点在于风险对齐和设计约束写得很系统，适合作为任务分析层的框架。

方法缺陷

它对人工知识整理依赖较强，容易把建模成本前移到分析阶段。另一个问题是，它更偏方法论和决策框架，直接替换现有分割网络的力度有限。

适合复现的理由

复现的关键不是训练技巧，而是把知识卡表述和映射规则实现清楚，因此代码难度中等。数据集公开，评价指标明确，适合作为“任务分析 + 模型选择”范式来复现。

适合改造的理由

最适合改造的是眼底分割的数据集分析流程，可以帮助你判断是否需要更强的边界建模、跨域泛化或提示式交互。我的判断是，它适合和 XSSR、WaveSDG 这类方法结合，形成“数据-策略-模型”三段式设计。

5. Quality-Guided Semi-Supervised Learning for Medical Image Segmentation

背景

论文明确指出，半监督分割常依赖伪标签，但常见做法只看模型置信度或不确定性，这不等于真实 mask 质量。对眼底分割来说，这个问题很现实，因为高噪声伪标签会直接污染血管细线和边界。

模型 / 算法

方法先用合成 corruption 和不成熟分割器输出构造 variable-quality masks，再训练 quality predictor 估计 image-mask pair 的质量，随后把该预测器接入质量感知正则项和伪标签重加权。流程图清楚表达了“先评估质量，再使用质量”的两阶段思路。

训练方法

论文明确把 quality predictor 设计成可作为已有 SSL 框架的 drop-in 增强模块。也就是说，它不是要求你重写整个半监督系统，而是补一个质量判别器就能改变伪标签使用方式。

数据来源

6. Med-DisSeg: Dispersion-Driven Representation Learning for Fine-Grained Medical Image Segmentation

背景

论文明确指出，医学分割的很多失败不是“看不到目标”，而是 encoder 的表征塌缩和 decoder 对细粒度边界恢复不足。对眼底来说，这个问题会直接影响小血管、毛细血管和边界过渡区。

模型 / 算法

方法在编码端加入 lightweight dispersive loss，把 batch 内 hidden representations 当作 negative pairs 拉开间隔；解码端再用 adaptive multi-scale calibration 保留局部纹理和全局形状。流程图说明它是“先让表示更分散，再让 decoder 做多尺度修复”。

训练方法

论文明确强调 loss 额外开销很小，主要是 boundary-aware embedding 和 fine-grained decoding 的组合。训练逻辑很像把表征层和恢复层分工，避免 decoder 直接承担全部细节恢复压力。

数据来源

论文覆盖五个数据集、三种模态，但未在可见文本中完整列出所有名称。信息缺失处只能写未明确说明，不过实验覆盖面足以说明它不是单任务方法。

具体创新点

它先在表示空间上阻止塌缩，再让 decoder 做多尺度修复。我的判断是，这种设计对眼底小血管特别友好，因为细结构任务往往不是缺容量，而是缺可分辨的边界表示。

实验设计

论文在 5 个数据集和 3 种模态上做 SOTA 对比，并报告跨任务鲁棒性。摘要未展开具体消融表名，但从分析段看，核心验证集中在 dispersive loss 和多尺度校准这两个部分。

实验结果

论文明确写到在 5 个数据集、3 种模态上都有竞争力，并声称跨任务鲁棒性较好。可见文本没有逐项列出全部数值，因此这里以作者总体结论为准。

局限

摘要没有细化到眼底任务，因此对超薄血管的专门收益还需要单独验证。我的判断是，它的效果很可能依赖数据分布，真正迁移到 fundus 时仍要看是否会引入不必要的表示分散。

对眼底分割的可用性

这篇特别适合迁移到细血管、边界保真和小目标恢复场景。落地时最可能的用法，是把 dispersive loss 和现有 U-Net / Transformer decoder 结合，作为细粒度增强项。

审稿式判断

方法优点

论文明确把“表征分散”和“多尺度恢复”拆开处理，这个分工对细结构任务很合理。我的判断是，它的优点在于额外开销相对小，同时逻辑上能闭合到边界保真问题。

方法缺陷

它不是专门为眼底设计的，因此对超薄血管和复杂背景的收益并不确定。另一个问题是，“分散化”并不总是越强越好，过度拉开表征也可能伤害细粒度一致性。

适合复现的理由

方法模块简单，主要就是一个损失项加一个多尺度解码策略，代码实现难度不高。数据集多模态但接口通用，因此适合先在眼底数据上做小规模验证。

适合改造的理由

它很适合插入眼底分割的 encoder-decoder 主干里，用作边界和小血管增强模块。我的判断是，后续如果想专门攻细血管连续性，这会比单纯堆注意力更稳一些。

领域进展速览

  1. 轻量化/部署方面，今天最相关的是 MS-DKC 和 XSSR：前者从任务知识出发约束模型选择，后者在固定标注预算下优化样本选择，二者都更接近“少资源可落地”的实际需求。
  2. 细血管或边界保真方面，Effect of Input Resolution 和 Med-DisSeg 最值得看：前者证明分辨率会真实伤害 thin vessel sensitivity，后者则给出一种把细粒度边界从表示层往回修复的方法。
  3. 跨域/泛化方面，Cross-Domain Vessel Segmentation、XSSR 和 MS-DKC 都在强调“不是单一模型大小问题”，而是域偏移、任务约束和数据证据如何共同决定最终性能。
  4. 标注效率/提示/伪标签质量方面，XSSR 和 Quality-Guided Semi-Supervised Learning 最直接：一个解决“标哪些样本”，一个解决“哪些伪标签值得信”，都很适合继续迁移到眼底分割的低标注场景。

在多个未见目标域上持续取得更好的 balanced Dice 和更低 HD95，尤其对 OC 分割更稳，结构边界更清晰。

局限

这是单源泛化设定，仍依赖训练源域覆盖度；wavelet 结构带来额外设计复杂度，跨任务验证还不够宽。

对眼底分割的可用性

对眼底 OD/OC 的跨设备迁移最直接，适合后续作为鲁棒性基线。

审稿式判断

方法优点

论文把 wavelet 子带按结构、边缘和噪声语义去分解，方向非常明确。我的判断是，它最大的优点是把单源域泛化里最难的“外观变化”拆成了可操作的频域组件。

方法缺陷

它仍然依赖源域覆盖度，且 wavelet 设计增加了模型的结构复杂性。我的判断是，如果目标域偏移超出源域多样性，单靠这个模块还不够。

适合复现的理由

源域和多个未见目标域都公开可获取，WISER 的设计边界清晰，比较适合复现。我的判断是，它特别适合作为眼底跨设备泛化的基线增强。

适合改造的理由

WISER 很适合被改造成更通用的边界/结构分离模块。我的判断是，如果后续做 OD/OC 泛化或跨相机鲁棒性，这一套设计很值得继续延伸。

审稿式判断

方法优点

论文把跨域一致性和域内伪标签学习放进一个统一框架，teacher-student 结构也比较稳。我的判断是，它对 OD/OC 这种跨中心部署问题给出了一条完整、可讲清楚的 UDA 路线。

方法缺陷

标准 UDA 仍然依赖源域可访问，且不同子模块之间的平衡会影响最终表现。我的判断是，这类方法最难的地方不是有没有提升，而是提升是否在所有域上都稳定。

适合复现的理由

公开数据和对比方法都比较齐全，整体结构也清晰，适合复现主结果。我的判断是，它是眼底跨域适配里比较标准的一条 teacher-student 基线。

适合改造的理由

动态 mask、频域监督和伪标签学习都可以拆出来注入现有眼底分割流水线。我的判断是，它很适合继续做跨中心、跨设备和弱监督增强。

  • 论文 ID: 2606.08749v1
  • 归类: 可迁移
  • 保留理由: 开放集、源数据不可用时的主动适配

背景

临床迁移时常见问题是：源数据不可用，目标域还可能出现源域没有的新类别。

模型 / 算法

ASFOSDA 以 active learning 选择目标样本；CDU 评估 class-aware uncertainty，CPD 用 prototype discrepancy 选代表性样本；再用 target-refined self-training。

训练方法

在 MSD Spleen、FLARE、AMOS CT、AMOS MR 等 3D 分割任务上训练；使用 3D U-Net 和 SwinUNETR，Dice+CE 损失，source 预训练后在 target 侧自适应。

数据来源

在 MSD Spleen、FLARE、AMOS CT、AMOS MR 等 3D 分割任务上训练；使用 3D U-Net 和 SwinUNETR，Dice+CE 损失，source 预训练后在 target 侧自适应。

具体创新点

同时解决 source-free 和 open-set 两个难点，用“先挑样本，再伪标注”的策略减少标注需求。

实验设计

做三套数据上的主动查询、伪标签和消融分析，并和 Source-only、ASFDA 等基线比较。

实验结果

整体优于现有方法，作者报告 U-Net 和 Swin UNETR 在主动自适应后能达到接近上界的性能比例。

适合复现的理由

适合改造的理由

7. Enhancing MedSAM with a Lightweight Box Predictor for Medical Image Segmentation

  • 论文 ID: 2606.04705v1
  • 归类: 可迁移
  • 保留理由: SAM / MedSAM 的提示增强，适合提示不稳定的眼底目标

背景

SAM/MedSAM 在医学图像上常被 prompt 质量限制，单点提示对小、弱对比、边界模糊目标尤其不稳。

模型 / 算法

在 MedSAM 上加轻量 Box Predictor，从单点估计粗框，再把框作为额外空间先验输入；Box Predictor 只增加约 1.6M 参数。

训练方法

Box Predictor 先单独训练，再与 MedSAM 组合；在 FLARE22、BRISC、BUSI、LungSegDB 上验证。

数据来源

Box Predictor 先单独训练，再与 MedSAM 组合；在 FLARE22、BRISC、BUSI、LungSegDB 上验证。

具体创新点

把 prompt 从“单点”升级为“点到框”的两阶段空间引导，减少 prompt 歧义。

实验设计

比较 point、box、hybrid 及冻结/微调配置，并做点偏移、框扩张比例和 patch size 消融。

实验结果

相对 baseline MedSAM 在多数据集上提升稳定性和精度，对 off-center click 更鲁棒。

局限

作者明确指出，单点仍然无法表达多个离散目标；当目标本身非常清晰时，额外框反而可能带来干扰。

对眼底分割的可用性

对眼底里“点提示不够准”的交互式分割很有迁移价值，但更偏方法底座。

领域进展速览

  1. 轻量化仍然是眼底血管分割的重要方向，尤其适合移动筛查和边缘设备。
  2. 局部低对比、细末梢断裂和边缘不连续，仍然是 OCTA 和常规 fundus 的主难点。
  3. OD/OC 分割的趋势已经从“单域最好”转向“跨域稳定”，WaveSDG 和 DDS-UDA 是这一变化的代表。
  4. SAM / MedSAM、active learning、source-free / open-set adaptation 的价值在于补齐提示、标注和域迁移三个薄弱环节。
