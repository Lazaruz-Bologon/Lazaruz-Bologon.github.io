其余当天新条目主要是遥感、动作识别、视频编辑、工业异常检测、通用 VLM 评估或非分割任务；这些论文要么不直接涉及医学图像分割，要么虽含 medical 关键词但对眼底分割的可迁移价值太弱，因此未进入正式列表。

总体判断

  1. 直接面向眼底分割的主线仍然围绕细结构连续性、噪声鲁棒性和边界修复展开，CurvSegFlow 的 flow matching + 迭代细化非常贴近血管分割；BAC-JEPA 也从“低标注 + 合成监督”角度提供了可迁移路线。
  2. 可迁移方法的主线是 foundation model 低参数适配和训练配方增强，EffiCell-Seg 说明冻结 encoder 后仍可靠结构先验拿到稳定收益，DP-Morph 则说明训练策略与结构修复可以在隐私约束下继续加分。
  3. 后续最值得跟进的是“结构先验 + 细边界修复 + 低标注/隐私训练”的组合，而不是单纯扩大 backbone；如果要落地到眼底分割，优先考虑 CurvSegFlow 类 refinement、EffiCell-Seg 类 VFM adapter 和 DP-Morph 类训练配方。

逐篇精读

CurvSegFlow: Time-Conditioned Flow Matching for Robust Segmentation of Curvilinear Structures in Noisy Biomedical Images

  • 论文 ID: 2606.21608v1
  • 原始链接: https://arxiv.org/abs/2606.21608
  • 归类: 直接相关
  • 相关性判断: 论文明确针对生物医学曲线结构分割，直接包含视网膜血管等基准。
  • 为什么保留: 直接覆盖细结构连续性、低信噪比和跨模态泛化，和眼底血管分割的痛点高度一致。

背景

论文明确指出，曲线状结构在生物医学图像中常受细长几何、拓扑复杂度和噪声影响，尤其微管和视网膜血管容易断裂。作者把分割重写为“从噪声初始化逐步细化到目标结构”的动态过程，而不是单次前向预测。

模型 / 算法

模型是 U-Net backbone + time-conditioned flow matching。输入由原图与噪声初始化 mask 组成，随时间步通过学习到的 velocity field 逐步修正分割结果；作者还引入 temporal embeddings 和 triple-term loss。

训练方法

训练细节在论文中以训练协议形式给出，核心是围绕 reconstruction stages 做逐步细化训练。未明确说明统一的公开代码训练脚本，但文中明确区分了合成微管、真实微管和其他曲线结构三类实验。

数据来源

数据来源包括 MicSim_FluoMT-Simple、MicSim_FluoMT-Complex、MicReal_FluoMT、Revised Higaki 2024、IRM_InVitroMT，以及 DRIVE、CHASE_DB1、CORN-1、ARCADE。图1明确展示了这些数据集的多模态样例和对应标注。

具体创新点

具体创新点是把 flow matching 用于分割的“渐进纠错”而非生成任务，强调连续性修复和低信噪比下的误差抑制。图2给出了整体框架，说明噪声初始化、时间条件和三项损失如何一起工作。

实验设计

实验按微管分割和其他曲线结构分两大部分展开，且设有 ablation studies。论文明确把 Table 1-5 分别用于不同数据集结果，Figure 5-12 用于定性结果与跨数据集比较。

实验结果

在 MicSim_FluoMT-Simple 上，CurvSegFlow 总体最好，Dice 0.926 左右，略优于 nnU-Net；在 MicSim_FluoMT-Complex 上，它取得最高 Dice、MCC 和 AUC-PR；在 MicReal_FluoMT 上，CurvSegFlow Dice 0.7093，优于 nnU-Net 的 0.7040。对 DRIVE 和 CHASE_DB1 也保持了顶级或接近顶级表现，Table 4/5 显示其 Dice、IoU、AUC-ROC 竞争力很强。

局限

论文自己承认仍主要在曲线结构上验证，且真实跨域、更多器官和更复杂病灶的外推还需要继续检验。我的判断是，flow-based 迭代虽然稳定，但训练链路比普通 U-Net 更复杂。

对眼底分割的可用性

对眼底分割最可用的场景是血管、毛细血管、边界连续性要求高的结构，尤其低对比、噪声重或需要保持连通性的任务。适合改造成眼底血管分割的 refinement head 或结构一致性模块，落地难度中等偏高，但方法闭环很强。

审稿式判断

  • 方法优点：论文明确把细结构连续性作为主目标，方法和数据集选择高度一致，Figure 2 与 Tables 1-5 形成完整闭环。跨微管、视网膜血管、角膜神经和冠脉的结果支持了“不是只对单一数据集有效”的主张。
  • 方法缺陷：flow matching + 时间嵌入 + 三项损失带来更复杂的训练/推理链路，且作者主要证明了曲线类结构上的收益。我的判断是，如果转到眼底分割里的大面积区域，收益可能不如细血管任务稳定。
  • 适合复现的理由：公开页面给出了完整架构图、数据集划分和多张结果表，主干还是 U-Net 类结构，工程上可控。数据集多数公开，至少可先在 DRIVE/CHASE_DB1 上重现定性和定量趋势。
  • 适合改造的理由：可以直接把 flow refinement 模块插入现有眼底分割流水线，用于边界修复、断裂血管补全和伪标签后处理。也适合和小血管专门损失、中心线约束或 topology-aware loss 组合。

Rethinking the Adaptation of Vision Foundation Models for Efficient Cell Segmentation

  • 论文 ID: 2606.21913v1
  • 原始链接: https://arxiv.org/abs/2606.21913
  • 归类: 直接相关
  • 相关性判断: 论文明确做细胞分割中的 VFM 适配，属于可直接迁移的医学分割方法。
  • 为什么保留: 核心是冻结视觉基础模型、抽取结构先验、降低训练参数量，这类改法很适合迁移到眼底分割的小样本或跨域设定。

背景

论文指出，细胞分割依赖昂贵标注和重型微调，而视觉基础模型虽然泛化强，但直接重训 encoder 代价过高。作者把问题定义为“如何从冻结的 VFM 中提炼结构先验”。

模型 / 算法

模型由 CSP-Encoder 和 SM-Decoder 组成。CSP-Encoder 从冻结 VFM 特征中合成语义显著图和形态特征，SM-Decoder 再联合预测几何距离场和语义 map，并用 cross-guidance 强化一致性。

训练方法

训练时尽量不重训视觉编码器，trainable 参数仅约 4.99M。实验设置对 CellSeg 和 DSB 两个数据集做 semantic segmentation 和 instance segmentation，并在自动提示模式下评估。

数据来源

数据集是 CellSeg 与 DSB。Table 1 和 Table 2 明确报告了两个数据集上的 semantic / instance 结果，Figure 1-2 解释了结构先验图的来源和整体框架。

具体创新点

具体创新点是把 VFM 的全局显著性和局部形态信息拆成两种结构先验，再用提示编码器和掩码解码器闭环组合起来。Table 3 的消融证明两个先验和 decoder 都有独立贡献。

实验设计

实验包含与 CPPNet、Swin-UMamba、CellSAM、PromptNucSeg、CellViT、SAC 的对比，以及完整消融。作者用 Dice、mIoU、PQ 和 F1 作为主指标，并明确统计 trainable 参数量。

实验结果

Table 1 显示 EffiCell-Seg 在 CellSeg 上 Dice 87.70、mIoU 78.48，在 DSB 上 Dice 93.14、mIoU 87.37，均优于对比方法。Table 2 显示 instance segmentation 的 PQ/F1 也同步领先，同时只需约 4.99M 可训练参数。

局限

论文自己没有解决 VFM 预训练偏置问题，也没有证明该方案对非常弱纹理或超大视野任务一定成立。我的判断是，冻结 encoder 虽省算力，但遇到域差过大时上限可能受限。

对眼底分割的可用性

对眼底分割很有用，尤其适合把 SAM/VFM 改造成低参数、可微调的 retina adapter。适合替换现有 encoder 或加一个结构先验分支，用于小样本、跨医院、跨设备迁移。

审稿式判断

  • 方法优点：结构先验拆解得清楚，CSP-Encoder 和 SM-Decoder 的职责边界明确，Table 3 的消融闭环完整。结果在两个细胞数据集上都稳定领先，且参数量明显更小。
  • 方法缺陷：依赖高质量 VFM 表征，若底层特征对眼底细小血管不敏感，性能上限会被卡住。我的判断是，方法更适合做适配层，不适合单独承担全部分割压力。
  • 适合复现的理由：架构清晰，核心就是冻结 backbone + 结构先验编码 + mask decoder，容易在现有代码里拼出来。数据集和指标也标准，比较容易重现实验趋势。
  • 适合改造的理由：可直接改成眼底分割的 VFM adapter，尤其适合引入血管显著性图、中心线先验或多尺度 morphology prior。也适合做跨模态迁移和轻量化部署。

From Gradient Clipping to Structural Refinement: Improving DPSGD for Medical Image Segmentation

  • 论文 ID: 2606.21763v1
  • 原始链接: https://arxiv.org/abs/2606.21763
  • 归类: 直接相关
  • 相关性判断: 论文明确研究隐私保护下的医学分割训练策略，属于训练配方层面的直接相关工作。
  • 为什么保留: 它解决的是“医学分割在隐私约束下怎么保性能”，对需要联邦或院内闭环训练的眼底场景很实用。

背景

论文指出，医疗分割模型接触敏感数据，DPSGD 可保护隐私但会牺牲效用。作者质疑分类任务里的 clipping 经验能否直接迁移到 segmentation。

模型 / 算法

方法比较了 Auto-S、NSGD、PSAC 等 clipping 策略，并在此基础上加入 morphological refinement，最终提出 adaptive DP-Morph。核心不是换主干，而是改训练和后处理。

训练方法

实验采用 OCT 的 Duke、UMN，以及 CT 的 COVID-19 数据集，分别覆盖多类和二类分割。论文还分析了 gradient alignment、global loss attack 和计算开销。

数据来源

数据来源明确是 Duke、UMN 和 COVID-19，Figure 1 展示了样本与标注，Figure 2 是 clipping 超参数热图，Figure 3-6 对比了不同策略与形态学修复。

具体创新点

具体创新点是把 clipping 策略和结构修复联合考察，并指出 PSAC 的某些分类假设在分割里并不稳定。论文明确强调了 morphology 在 privacy setting 下能抵消一部分性能退化。

实验设计

实验设计包括多种 clipping 策略对比、梯度分析、global loss attack、形态学修复与自适应修复的消融。作者用 Dice 和 HD95 作为主要分割指标。

实验结果

论文明确给出：形态学修复在非私有设置下让 UNet Dice 约提升 0.6%，UNet++ 提升约 0.5%，HD95 分别降低约 17% 和 19%。在 DPSGD 下，Auto-S 和 Flat 往往最好，DP-Morph 可把 Dice 再提升约 1-2%。

局限

作者自己承认，不同 clipping 策略在分割上的表现和分类结论并不完全一致，尤其是归一化类方法。我的判断是，实验主要覆盖 OCT/CT，跨到更细的眼底血管或视盘边界还要再验。

对眼底分割的可用性

对眼底分割可直接落到隐私训练场景，尤其是多医院联合训练、院内数据不可出域时。适合和现有分割主干搭配，作为训练配方与后处理增强，而不是替换主干。

审稿式判断

  • 方法优点：把隐私训练中最常见的 clipping 问题和结构修复绑定起来，Figure 3-6 给出很直接的实验闭环。结果不只是提升 Dice，也看到了 HD95 的改善，说明边界质量一起变好。
  • 方法缺陷：需要额外的 morphology 规则和超参数，且不同数据集的最优 clipping 组合并不统一。我的判断是，若任务本身结构很碎，手工形态学操作可能引入额外偏差。
  • 适合复现的理由：基线模型、数据集和评价指标都标准，clipping 策略也有明确名字，容易做对照实验。对眼底数据复现时可直接替换为 DRIVE/CHASE_DB1 或私有多中心数据。
  • 适合改造的理由：非常适合嵌入眼底隐私训练流水线，特别是联邦学习、DP 训练和伪标签后处理。也适合改造成血管/杯盘的 class-specific morphology refinement。

BAC-JEPA: Label-Efficient Breast Arterial Calcification Segmentation via Synthetic Mammography-Guided Supervision

  • 论文 ID: 2606.22089v1
  • 原始链接: https://arxiv.org/abs/2606.22089
  • 归类: 直接相关
  • 相关性判断: 论文明确做 label-efficient segmentation，而且用 synthetic supervision 缓解像素级标注成本。
  • 为什么保留: 虽然任务不是眼底，但 synthetic supervision + low-label transfer 的思路很适合迁移到视网膜病灶、血管或杯盘分割。

背景

BAC 是筛查乳腺片中的心血管风险生物标志物，但像素级标注昂贵。作者用合成钙化插入真实乳腺背景的方式，构造“有精确 mask 的合成监督”。

模型 / 算法

模型由 mammography self-supervised ViT encoder + high-resolution convolutional decoder 组成。合成 BAC 的生成器会采样血管结构、病灶负荷、影像外观和 hard negatives。

训练方法

训练使用 75,472 份乳腺检查做背景选择与表征学习，10,000 个训练背景生成合成数据，1,000 个 development 背景做 checkpoint 选择，并在 1,000 个 BacSeg 合成 mammograms 上评估迁移。

数据来源

数据来源在摘要和方法里写得很具体：75,472 studies、34,956 patients、10,000 training backgrounds、1,000 development backgrounds、1,000 BacSeg synthetic 2D mammograms。Figure 1-2 分别展示合成示例和真实输出。

具体创新点

创新点是把“背景筛选 + 合成插入 + 自监督 encoder + 高分辨率 decoder”串成一条低标注闭环。它不只是 data augmentation，而是用合成 pixel supervision 做 label-efficient training。

实验设计

实验包括 held-out synthetic validation、BacSeg image-level transfer、四视图推理耗时和合成生成耗时。作者明确报告了 IoU、Dice、AUROC 和推理时间。

实验结果

held-out synthetic validation 上更大 backbone 达到 IoU 0.5325、Dice 0.6357；在 BacSeg 上，基于 segmentation probability maps 的 image-level AUROC 为 0.8719，小 backbone 为 0.8547。四视图推理仅 110.68-213.63 ms，合成生成平均 2.7071 s/图。

局限

论文自己明确说，真实 mammogram 的 expert-reviewed segmentation 仍然是临床验证和校准所必需的。我的判断是，这类 synthetic supervision 对真实标签噪声和域偏移仍然敏感。

对眼底分割的可用性

对眼底分割的可用性在于低标注监督和合成病灶/血管插入的思路，可用于少标注视盘、微出血或渗出物分割。落地难度较高，但很适合作为数据扩增与预训练路线。

审稿式判断

  • 方法优点：数据生成、背景筛选、模型训练和迁移评估是一条完整链路，论证很闭环。结果不仅有像素级指标，也有 image-level transfer，说明方法的实用面不止一个。
  • 方法缺陷：强依赖合成器和背景筛选器的质量，若合成分布偏差大，迁移收益会缩水。我的判断是，它更像高成本数据工程方案，不是轻量即插即用模块。
  • 适合复现的理由：论文给了明确的数据规模、生成流程和指标，尽管实现量大，但实验设计相当清楚。若只迁移思路到眼底，先做小规模 synthetic lesion segmentation 也可验证。
  • 适合改造的理由：非常适合迁移到眼底的合成病灶生成、弱标注训练和预训练数据构造。也适合做 vessel-like object 的 synthetic supervision 和 pseudo-label 校准。

Native space based pipelines outperform template space based pipeline in subcortical segmentation

  • 论文 ID: 2606.21463v1
  • 原始链接: https://arxiv.org/abs/2606.21463
  • 归类: 可迁移
  • 相关性判断: 虽然不是眼底任务，但它讨论了 template space 与 native space 的取舍，对跨模态分割流水线有迁移价值。
  • 为什么保留: 它的核心问题是“在原空间做分割是否比模板空间更可靠”，这类空间归一化判断对眼底图像标准化也有启发。

背景

作者指出，很多亚皮层分割依赖模板空间配准，但这可能损伤个体特异的解剖边界，尤其对小结构不利。

模型 / 算法

他们开发了两个基于 UNet 的 STN/RN/SN 分割管线，并比较 native-space 与 template-space 的表现，还测试模板分辨率和 7T->3T 迁移。

训练方法

使用 7T 和 3T MRI，来自 5 个 public datasets；在 held-out 7T 上比较 native 与 template pipeline，并进一步测试 7T 训练模型迁移到 3T。

数据来源

数据来源是 5 个 public datasets 的 7T/3T MRI。论文摘要明确提到 STN、RN、SN 三个结构以及手工标签作为评估标准。

具体创新点

具体创新点是把 native-space 与 template-space pipeline 做了直接对照，并用更高分辨率模板和 synthetic 3T 数据测试“模板是否真的有帮助”。

实验设计

实验设计包含 held-out 7T 对照、template resolution 变化、7T->3T transfer 和 synthetic 3T augmentation。指标用 Dice 和 HD95，主要看边界几何 fidelity。

实验结果

在 held-out 7T 上，native pipeline 一贯优于 template pipeline，例如 STN Dice 0.775±0.055 对 0.713±0.051，HD95 0.79±0.24 mm 对 1.17±1.10 mm。模板分辨率提高并没有带来明显收益，3T 上所有模型都掉点。

局限

作者自己指出 7T-to-3T 的域差仍然是开放问题，synthetic 3T 只带来 modest improvements。我的判断是，这说明空间规范化不是万能补丁。

对眼底分割的可用性

对眼底分割的可用性在于提醒我们不要过度依赖模板空间或强配准，尤其是个体化结构边界。适合迁移到 OCT、视盘杯盘和小病灶分割的空间预处理决策。

审稿式判断

  • 方法优点：问题定义清晰，native vs template 的比较直接，结果对临床个体化边界很有说服力。它不是纯架构改进，而是在流水线层面给出可执行结论。
  • 方法缺陷：任务和模态与眼底不同，外推要谨慎。我的判断是，它给的是“空间处理原则”，不是可直接复用的模型组件。
  • 适合复现的理由：基线清楚，UNet 管线和评价指标都标准，复现实验逻辑不复杂。即便数据获取门槛较高，思路本身容易在其他分割任务里复测。
  • 适合改造的理由：可以把 native-space 优先的思路迁移到眼底，不必过度模板化 preprocessing。对 retina、cup-disc 及跨设备域适配都值得试。
